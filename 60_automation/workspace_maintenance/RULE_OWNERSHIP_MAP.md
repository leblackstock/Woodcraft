## Classification Rule

- Canonical owner: update the owner first, then repair pointers.
- Intentional standalone copy: retain complete context that an external target or separate agent tool cannot read from the repository.
- Short pointer or summary: keep it brief and link to the owner.
- Stale or conflicting duplicate: remove, replace with a pointer, or archive when its successor is known.
- Needs user decision: escalate factual, policy, published-record, source-evidence, or ownership uncertainty.
